# 三、容器与编排

## 2.1 Docker

### Dockerfile 要点

```dockerfile
FROM node:18-alpine AS builder
WORKDIR /app
COPY package*.json ./
RUN npm ci --only=production

FROM node:18-alpine
WORKDIR /app
COPY --from=builder /app/node_modules ./node_modules
COPY . .
EXPOSE 3000
CMD ["node", "server.js"]
```

| 技巧 | 说明 |
|:----|:------|
| **多阶段构建** | 构建与运行分离，镜像体积大减 |
| **Alpine 基础镜像** | 最小化攻击面 |
| **.dockerignore** | 排除 node_modules、.git 等无关文件 |
| **layer 缓存** | 先 COPY 依赖描述文件再 RUN install，利用缓存 |

### docker-compose

```yaml
version: "3.8"
services:
  app:
    build: .
    ports: ["3000:3000"]
    environment:
      DB_HOST: db
    depends_on: [db]
  db:
    image: postgres:16-alpine
    volumes: [pgdata:/var/lib/postgresql/data]

  pgdata:
```

### 镜像安全扫描

```bash
# Trivy 扫描
trivy image myapp:v1.2.3

# 构建时集成
docker build --build-arg TRIVY_SCAN=true -t myapp:v1.2.3 .
```

## 2.2 Kubernetes

### 核心资源

| 资源 | 作用 |
|:----|:------|
| **Pod** | 最小调度单位 |
| **Deployment** | 无状态应用，声明式副本控制 |
| **StatefulSet** | 有状态应用（数据库、消息队列） |
| **Service** | 服务发现与负载均衡 |
| **Ingress** | HTTP 路由转发（域名 → Service） |
| **ConfigMap** | 非敏感配置 |
| **Secret** | 密码、Token、证书 |
| **PVC** | 持久化存储申请 |

### Deployment 示例

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: myapp
spec:
  replicas: 3
  selector:
    matchLabels:
      app: myapp
  template:
    metadata:
      labels:
        app: myapp
    spec:
      containers:
      - name: app
        image: myapp:v1.2.3
        ports: [{containerPort: 3000}]
        resources:
          requests: {cpu: 100m, memory: 128Mi}
          limits: {cpu: 500m, memory: 256Mi}
        livenessProbe:
          httpGet: {path: /health, port: 3000}
        readinessProbe:
          httpGet: {path: /ready, port: 3000}
```

### Service + Ingress

```yaml
# Service
apiVersion: v1
kind: Service
metadata:
  name: myapp
spec:
  selector: {app: myapp}
  ports: [{port: 80, targetPort: 3000}]
---
# Ingress
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: myapp
spec:
  ingressClassName: nginx
  rules:
  - host: app.example.com
    http:
      paths:
      - path: /
        pathType: Prefix
        backend:
          service: {name: myapp, port: {number: 80}}
```

## 2.3 调度与资源

| 概念 | 说明 |
|:----|:------|
| **requests** | 调度保证量，决定 Pod 落到哪个节点 |
| **limits** | 运行上限，超出 CPU 被限流，内存被 Kill |
| **亲和性** | 指定 Pod 与哪些 Pod 靠近 / 远离 |
| **污点与容忍** | 节点打 taint，Pod 加 toleration 才能调度上去 |
| **HPA** | 横向自动伸缩，基于 CPU/内存/自定义指标 |
| **QoS** | Guaranteed / Burstable / BestEffort |

## 2.4 网络与存储

| 模块 | 常用方案 |
|:----|:------|
| **CNI** | Calico（网络策略强）/ Flannel（简单） |
| **CSI** | 块存储 / NFS / 对象存储 |
| **NetworkPolicy** | 精细控制 Pod 间通信 |

## 2.5 Helm

```bash
# 创建 Chart
helm create myapp

# 目录结构
myapp/
├── Chart.yaml
├── values.yaml
├── templates/
│   ├── deployment.yaml
│   ├── service.yaml
│   └── ingress.yaml

# values 分层
helm install myapp ./myapp \
  -f values-base.yaml \
  -f values-prod.yaml
```

| 概念 | 说明 |
|:----|:------|
| **Chart** | Helm 包，含模板 + values |
| **values 分层** | base → env(dev/test/prod) → override，逐层覆盖 |
| **Release** | Chart 的一次安装实例 |
| **Repo** | Harbor / Chart Museum 托管和分发 |

## 2.6 多集群

| 场景 | 方案 |
|:----|:------|
| **跨地域灾备** | 多集群部署，DNS/GSLB 分流 |
| **跨集群发布** | ArgoCD + ApplicationSet 管理多集群同步 |
| **集群联邦** | KubeFed 跨集群资源同步 |
| **混合云** | 部分业务在自建 K8s，部分在公有云 |
