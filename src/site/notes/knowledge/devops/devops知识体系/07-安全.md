# 七、安全

> 安全不是最后一环，应该从需求阶段就嵌入 DevOps 流程——这就是 DevSecOps。

## 7.1 DevSecOps 核心理念

```
传统：  开发 → 测试 → 安全（最后一刻）→ 上线
DevSecOps：安全贯穿全程
```

| 阶段 | 安全动作 |
|:----|:------|
| **设计** | 威胁建模（STRIDE）、安全需求评审 |
| **开发** | 安全编码规范、IDE 安全插件、Commit 扫描 |
| **构建** | SAST、依赖漏洞扫描、镜像扫描 |
| **测试** | DAST、渗透测试、模糊测试 |
| **部署** | 镜像签名校验、策略即代码（OPA） |
| **运行** | 运行时安全、入侵检测、WAF |

## 7.2 供应链安全

### 软件物料清单 (SBOM)

```bash
# 生成 SBOM
syft myapp:v1.2.3 -o cyclonedx-json > sbom.json

# 扫描 SBOM 中的已知漏洞
grype sbom.json
```

| 框架 | 说明 |
|:----|:------|
| **SLSA** | Google 提出的供应链安全等级（L1-L4） |
| **Sigstore** | 无密钥签名和验证（Cosign） |
| **in-toto** | 构建全链路的完整性验证 |

### 镜像签名

```bash
# 签名
cosign sign --key cosign.key myapp:v1.2.3
# 验证
cosign verify --key cosign.pub myapp:v1.2.3
# 部署策略：只允许签名镜像
kubectl create secret generic cosign-pub --from-file=cosign.pub
```

### 依赖扫描

| 工具 | 语言 | CI 集成 |
|:----|:------|:------|
| `npm audit` | JavaScript | 内置 |
| `pip-audit` | Python | GitHub Actions |
| `OWASP Dependency-Check` | Java | Jenkins / Maven 插件 |
| `trivy fs` | 通用 | 扫描源码目录的依赖文件 |

## 7.3 密钥管理

### Vault — 企业级密钥管理

```
App 启动 → 从 Vault 动态获取数据库凭据（有效期 1h）
  → 到期自动失效，App 重新获取
  → 无需把密码写死在 ConfigMap 里
```

| 功能 | 说明 |
|:----|:------|
| **静态密钥** | KV Store，加密存储 API Key、Token |
| **动态密钥** | 按需生成临时数据库凭据，到期自动回收 |
| **加密即服务** | 应用无需自己管理加密逻辑 |
| **PKI** | 内部 CA，自动签发和轮换 TLS 证书 |

### 密钥管理原则

| 原则 | 说明 |
|:----|:------|
| **最小权限** | 每个密钥只给必需的权限 |
| **定期轮换** | 所有凭据设有效期，到期自动更换 |
| **不硬编码** | 禁止密钥出现在代码、配置文件、镜像中 |
| **审计** | 记录所有密钥的访问和使用情况 |

## 7.4 访问控制

### RBAC

```yaml
# K8s RBAC：允许只读 Pod
apiVersion: rbac.authorization.k8s.io/v1
kind: Role
metadata: {name: pod-reader, namespace: production}
rules:
- apiGroups: [""]
  resources: ["pods"]
  verbs: ["get", "list"]
```

### 堡垒机

| 功能 | 说明 |
|:----|:------|
| 统一入口 | 所有 SSH/RDP 操作经过堡垒机，不直连 |
| 操作审计 | 命令录制、会话回放 |
| 权限管控 | 谁能在什么时间访问什么机器 |
| 国产推荐 | JumpServer |

### 身份认证

| 协议 | 说明 |
|:----|:------|
| **LDAP** | 企业目录服务（Windows AD / OpenLDAP） |
| **OIDC/OAuth2** | 现代单点登录（SSO） |
| **SAML** | 企业级 SSO |

## 7.5 运行时安全

| 工具 | 场景 |
|:----|:------|
| **Falco** | K8s 运行时异常行为检测 |
| **OPA / Gatekeeper** | 策略即代码，阻止不合规部署 |

## 7.6 合规

| 标准 | 适用场景 |
|:----|:------|
| **SOC2** | SaaS 服务安全认证 |
| **ISO 27001** | 信息安全管理系统 |
| **等级保护** | 国内合规强制要求 |
| **PCI DSS** | 支付卡行业数据安全 |
| **GDPR** | 欧盟数据保护法规 |

## 7.7 安全 Checklist

- [ ] CI 流水线集成 SAST + 依赖扫描
- [ ] 镜像推送前 Trivy 扫描，高危阻断
- [ ] 所有生产镜像签名验证
- [ ] 密钥使用 Vault 管理，不硬编码
- [ ] 访问使用堡垒机 + RBAC
- [ ] 生产 K8s 启用 Pod Security Standards
- [ ] 定期渗透测试和安全审计
