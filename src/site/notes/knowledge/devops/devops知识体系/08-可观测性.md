# 八、可观测性

## 5.1 三大支柱

| 支柱 | 回答 | 工具 |
|:----|:------|:------|
| **指标 (Metrics)** | 系统是否正常？ | Prometheus + Grafana |
| **日志 (Logging)** | 出了什么问题？ | ELK / Loki / Splunk |
| **链路 (Tracing)** | 问题出在哪个环节？ | Jaeger / OpenTelemetry |

## 5.2 Prometheus — 指标监控

### 架构

```
被监控目标（Exporter / App）
    ↓ HTTP Pull（每 15s）
Prometheus Server
    ↓ PromQL 查询
Grafana（可视化）
    ↓ 告警规则
AlertManager（告警分发）
```

### 核心指标类型

| 类型 | 说明 | 示例 |
|:----|:------|:------|
| **Counter** | 只增不减 | HTTP 请求总数、错误总数 |
| **Gauge** | 可增可减 | CPU 使用率、内存使用量 |
| **Histogram** | 分布统计 | 请求延迟 P50/P95/P99 |
| **Summary** | 客户端分位数 | 同上，但客户端计算 |

### PromQL 常用查询

```promql
# 错误率
rate(http_requests_total{status=~"5.."}[5m])

# P99 延迟
histogram_quantile(0.99, rate(http_request_duration_seconds_bucket[5m]))

# CPU 使用率
100 - (avg(rate(node_cpu_seconds_total{mode="idle"}[5m])) * 100)
```

## 5.3 Grafana — 可视化

| 要素 | 说明 |
|:----|:------|
| **Data Source** | 对接 Prometheus / Loki / Elasticsearch 等 |
| **Dashboard** | 一组 Panel 的集合，按业务/服务分组 |
| **Panel** | 单个图表（时序图、柱状图、饼图、仪表盘） |
| **Alert** | 基于 Panel 数据设置告警规则 |
| **Variable** | Dashboard 级别的动态筛选（集群、命名空间、服务名） |

## 5.4 日志管理

| 方案 | 组成 | 适用 |
|:----|:------|:------|
| **ELK** | Elasticsearch + Logstash + Kibana | 传统部署、海量日志 |
| **Loki** | Loki + Promtail + Grafana | K8s 原生，与 Prometheus 统一查询 |
| **Splunk** | 商业产品 | 企业级合规需求 |

### 日志采集架构

```
应用容器（stdout/stderr）
    ↓ Fluentd / Promtail 采集
日志存储（Elasticsearch / Loki）
    ↓ 检索 + 可视化
Kibana / Grafana
    ↓ 告警规则
AlertManager
```

### 日志规范

- 结构化日志（JSON 格式），方便检索和解析
- 关键字段：timestamp、level、service、trace_id、message
- 错误日志必须包含堆栈信息

```json
{
  "timestamp": "2026-07-19T10:00:00Z",
  "level": "ERROR",
  "service": "order-service",
  "trace_id": "abc123",
  "message": "payment timeout",
  "user_id": "u456"
}
```

## 5.6 告警体系

### 告警分级

| 级别 | 说明 | 响应要求 | 示例 |
|:----|:------|:------|:------|
| **P0 紧急** | 服务全面不可用 | 5 分钟内响应 | 网站 502 |
| **P1 严重** | 核心功能受损 | 15 分钟内响应 | 支付成功率下降 |
| **P2 一般** | 非核心功能异常 | 1 小时内响应 | 某管理页面报错 |
| **P3 提示** | 潜在风险 | 工作时间处理 | 磁盘 80% 告警 |

### AlertManager 配置示例

```yaml
groups:
  - name: app-alerts
    rules:
      - alert: HighErrorRate
        expr: rate(http_requests_total{status=~"5.."}[5m]) > 0.05
        for: 5m
        labels: {severity: p0}
        annotations:
          summary: "{{ $labels.service }} 错误率超过 5%"
```

### 告警治理

- **收敛**：相同告警合并，避免重复通知
- **静默**：维护窗口期抑制告警
- **升级**：P0 未确认则逐级上升（群消息 → 电话 → 上级）
- **值班轮转**：明确每个人负责的时间段
