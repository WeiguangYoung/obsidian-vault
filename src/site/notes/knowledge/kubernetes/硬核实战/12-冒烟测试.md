# 冒烟测试

本实验执行一系列验证，确保 K8s 集群正常工作。

## 数据加密

验证静态数据加密功能。

创建一个普通 Secret：

```bash
kubectl create secret generic kubernetes-the-hard-way \
  --from-literal="mykey=mydata"
```

查看 etcd 中该 Secret 的存储内容（十六进制）：

```bash
ssh root@server \
    'etcdctl get /registry/secrets/default/kubernetes-the-hard-way | hexdump -C'
```

```text
00000000  2f 72 65 67 69 73 74 72  79 2f 73 65 63 72 65 74  |/registry/secret|
00000010  73 2f 64 65 66 61 75 6c  74 2f 6b 75 62 65 72 6e  |s/default/kubern|
00000020  65 74 65 73 2d 74 68 65  2d 68 61 72 64 2d 77 61  |etes-the-hard-wa|
00000030  79 0a 6b 38 73 3a 65 6e  63 3a 61 65 73 63 62 63  |y.k8s:enc:aescbc|
00000040  3a 76 31 3a 6b 65 79 31  3a ...
```

etcd 键前缀 `k8s:enc:aescbc:v1:key1` 表明数据已通过 `aes-cbc` 加密。

## 部署测试

验证 Deployment 创建和管理功能。

部署 nginx：

```bash
kubectl create deployment nginx \
  --image=nginx:latest
```

查看 Pod 状态：

```bash
kubectl get pods -l app=nginx
```

```bash
NAME                     READY   STATUS    RESTARTS   AGE
nginx-56fcf95486-c8dnx   1/1     Running   0          8s
```

### 端口转发

```bash
POD_NAME=$(kubectl get pods -l app=nginx \
  -o jsonpath="{.items[0].metadata.name}")

kubectl port-forward $POD_NAME 8080:80
```

```text
Forwarding from 127.0.0.1:8080 -> 80
```

在另一终端验证：

```bash
curl --head http://127.0.0.1:8080
```

```text
HTTP/1.1 200 OK
Server: nginx/1.27.4
...
```

按 `Ctrl+C` 停止端口转发。

### 日志

```bash
kubectl logs $POD_NAME
```

```text
...
127.0.0.1 - - [06/Apr/2025:17:17:12 +0000] "HEAD / HTTP/1.1" 200 0 "-" "curl/7.88.1" "-"
```

### 执行命令

```bash
kubectl exec -ti $POD_NAME -- nginx -v
```

```text
nginx version: nginx/1.27.4
```

## Service 测试

验证 Service 暴露应用的功能。

创建 NodePort Service：

```bash
kubectl expose deployment nginx \
  --port 80 --type NodePort
```

> 本集群未集成云供应商，无法使用 LoadBalancer 类型。

通过 NodePort 访问：

```bash
NODE_PORT=$(kubectl get svc nginx \
  --output=jsonpath='{range .spec.ports[0]}{.nodePort}')

NODE_NAME=$(kubectl get pods \
  -l app=nginx \
  -o jsonpath="{.items[0].spec.nodeName}")

curl -I http://${NODE_NAME}:${NODE_PORT}
```

```text
Server: nginx/1.27.4
...
```
