# 🚗 奇瑞汽车 CICD工程师 — 技术学习路线

## 1. 嵌入式构建体系（核心）
- GCC交叉编译工具链：arm-none-eabi-gcc 使用
- Makefile 编写与调试
- CMake 构建系统：add_library、target_link_libraries、生成器表达式
- 链接脚本（.ld）：MEMORY / SECTIONS 布局、Bootloader 分区
- 静态库(.a)与动态库(.so)管理
- 构建产物：.elf → .bin / .hex，objcopy / objdump / size 工具链

## 2. CI/CD 流水线搭建
- GitLab CI：.gitlab-ci.yml 语法、stage/job/artifact/cache
- Jenkins：Pipeline as Code、Groovy 脚本、多分支流水线
- 流水线模板化与参数化构建
- 制品归档（Nexus / JFrog Artifactory / Harbor）

## 3. 代码质量门禁
- 静态分析：Cppcheck、SonarQube、MISRA-C 检查
- 单元测试：Google Test (GTest) 集成到 CI
- 代码覆盖率：gcov / lcov 报告生成
- 开源合规扫描：FOSSology / Black Duck

## 4. Docker 与构建环境管理
- Dockerfile 编写、构建节点镜像封装
- 多阶段构建、构建缓存优化
- docker-compose 管理构建依赖服务

## 5. 车规流程加分
- ASPICE 基础概念（过程评估模型）
- ISO 26262 功能安全基础
- 需求-代码-测试双向追溯
- HIL / SIL / CANoe 自动化测试集成

## 6. 脚本自动化
- Shell 脚本：构建失败自动分析、日志解析
- Python：CI/CD 工具开发、自动化报告生成
