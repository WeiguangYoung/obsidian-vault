# 🚗 奇瑞汽车 — 岗位总结 & 技术栈

---

## 📌 两个岗位总结

### 1️⃣ CICD工程师（旧JD）

面向汽车软件研发体系的CI/CD与工具链建设岗位。

**核心职责：**
- 搭建CI/CD流水线（代码构建 → 静态检查 → 单元测试 → 制品归档 → 版本发布）
- 嵌入式构建环境管理（交叉编译工具链、构建脚本、License、Docker镜像）
- 质量门禁接入（MISRA检查、SonarQube、覆盖率、开源合规）
- 对接HIL/SIL/CANoe等测试环境，实现自动刷写→测试→报告闭环
- 支撑ASPICE、功能安全等车规流程的追溯要求

**偏重：** 嵌入式软件构建、车规流程、质量门禁

---

### 2️⃣ DevOps开发工程师（新JD/WIP）

DevOps平台软件研发岗位，构建企业自有DevOps基础设施。

**核心职责：**
- DevOps平台研发，搭建CI/CD自动化工具链
- 部署维护研发系统（Jira/Git/Jenkins/SVN/Harbor/Nexus）
- 设计开发自有DevOps基础设施
- K8s/Docker容器化运维
- 监控系统设计（Grafana+Prometheus），保障系统稳定性

**偏重：** 平台开发、K8s运维、监控体系、自研基础设施

## 🔧 完整技术栈清单

### 基础
- Linux（文件系统/进程/网络/systemd/shell命令）
- Git / SVN（分支策略/MR流程/Git Hooks）

### CI/CD
- Jenkins（声明式/脚本式 Pipeline、Shared Libraries、K8s Agent）
- GitLab CI（.gitlab-ci.yml）
- Groovy 脚本
- 制品管理（Harbor / Nexus / Jfrog）

### 容器 & 编排
- Docker（Dockerfile/多阶段构建/镜像瘦身/安全扫描）
- K8s（Pod/Deployment/Service/ConfigMap/Helm/排障）
- Docker Compose

### 构建工具
- CMake（CMakeLists.txt/toolchain.cmake/交叉编译）
- Bazel（BUILD文件/增量构建）
- Make
- 交叉编译（ARM GCC/--sysroot/动态库静态库）

### 质量门禁
- GTest / CUnit 单元测试
- Cppcheck / SonarQube 静态分析
- gcov / lcov 覆盖率
- MISRA C/C++ / Coverity
- SBOM / Trivy 安全扫描

### 监控
- Prometheus（指标采集/PromQL/Alertmanager）
- Grafana（Dashboard/数据源）

### 自动化脚本
- Shell（grep/awk/sed/流程控制）
- Python（subprocess/yaml/json/requests）

### 汽车行业（了解即可）
- ASPICE / ISO 26262 / ISO 21434
- AUTOSAR CP/AP
- HIL / SIL / CANoe

## 🦐 一句话

> **CICD岗 = 嵌入式构建 + CI/CD + 质量门禁 + 车规流程**
> **DevOps岗 = CI/CD平台 + K8s运维 + 监控体系 + 自动化开发**
> 两者交集在 **CI/CD流水线 + Linux + 脚本开发**，差异在车规深度 vs K8s+监控广度
