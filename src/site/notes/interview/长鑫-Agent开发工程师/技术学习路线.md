## 📚 学习路线

### 🔥 第一优先级（必须掌握）

#### 1. Python 深度开发（AI 应用方向）
- **基础扎实**
  - 数据类型、列表/字典推导式
  - 面向对象设计（抽象类、Mixin、协议）
  - 装饰器、生成器、上下文管理器
  - 类型提示（Typing）与 Pydantic（重要！）
- **异步编程**
  - `asyncio`：事件循环、协程、Task
  - `aiohttp` / `httpx` 异步 HTTP 请求
  - 并发调用 LLM API（限流、重试、回退）
- **工程化**
  - pytest 测试（单元测试 + 集成测试）
  - logging 日志系统（结构化日志）
  - 配置管理（Pydantic Settings + dotenv）
- **数据处理**
  - Pandas 基础（数据分析）
  - JSON / YAML 处理
  - 文本分块（Text Splitting）

#### 2. AI Agent 架构
- **Agent 核心概念**
  - **LLM 作为核心大脑**：理解 LLM 的能力边界
  - **工具（Tool/Function Calling）**
    - OpenAI Function Calling 机制
    - 自定义工具注册与调用
    - 工具描述编写最佳实践
  - **记忆（Memory）**
    - 短期记忆（对话历史窗口）
    - 长期记忆（向量数据库 + RAG）
    - 记忆压缩与摘要
  - **规划（Planning）**
    - ReAct 模式（思考-行动-观察循环）
    - Chain-of-Thought（思维链）
    - Task Decomposition（任务分解）
    - Reflexion（自我反思）
- **Agent 框架**
  - **LangChain**：Chain / Agent / Tool 生态
  - **LangGraph**：有状态多步 Agent 工作流
  - **AutoGen**：多 Agent 协作框架
  - **CrewAI**：角色化 Agent 团队
  - 理解框架设计思路，不依赖框架也能自己写 Agent

#### 3. RAG（检索增强生成）
- **RAG 基础架构**
  ```
  用户提问 → 检索 → 增强（上下文拼接） → LLM 生成 → 回答
  ```
- **文档处理（Chunking）**
  - 固定长度分块（Token-based）
  - 语义分块（Semantic Chunking）
  - 递归字符文本分割器（RecursiveCharacterTextSplitter）
  - 段落/标题感知分块
  - 分块策略评估（查准率 vs 查全率）
- **嵌入（Embedding）**
  - 嵌入模型：text-embedding-3-small / bge-m3 / m3e
  - 嵌入向量的存储与索引
  - 批量嵌入（并发 + 限流）
- **向量数据库**
  - **Milvus**：大规模向量检索
  - **ChromaDB**：轻量级本地向量库
  - **FAISS**：Meta 开源的向量索引库
  - **PGVector**：PostgreSQL 向量扩展
  - 索引类型对比（IVF / HNSW）
  - 混合检索（向量相似度 + 关键词 BM25）
- **检索优化**
  - HyDE（假设文档嵌入）
  - Query Rewriting（查询重写）
  - Multi-Query（多查询扩展）
  - RAG Fusion（结果融合排序）
  - 检索重排序（Cross-Encoder Reranker）
- **高级 RAG**
  - Agentic RAG（Agent 自主判断何时检索）
  - Corrective RAG（纠正性检索）
  - Self-RAG（自我反思检索）

#### 4. Context Engineering（上下文工程）
- **Prompt Engineering**
  - 系统提示（System Prompt）设计
  - 少样本学习（Few-Shot）
  - 结构化输出约束（JSON Schema）
  - 角色扮演与上下文注入
- **上下文窗口管理**
  - Token 计数与预算
  - 上下文滑动窗口
  - 关键信息优先保留
  - 上下文压缩与摘要
- **工具系统优化**
  - 工具描述编写（清晰、准确、简洁）
  - 工具选择与路由（让 LLM 选对工具）
  - 工具调用失败处理与重试
  - 动态工具注册（根据场景加载）
- **性能优化**
  - 减少冗余上下文
  - 缓存策略（Prompt Cache / Semantic Cache）
  - 并发控制与限流
  - 延迟优化（流式输出）

#### 5. Agent 评估体系
- **评估维度**
  - **准确率**：回答正确性
  - **召回率**：信息完整度
  - **幻觉率**：事实一致性
  - **延迟**：端到端响应时间
  - **成本**：Token 消耗
- **评估方法**
  - 人工评测（标注数据集）
  - LLM-as-Judge（用 LLM 评估 LLM）
  - 自动化评测（G-Eval / ROUGE / BLEU）
  - A/B 测试（版本对比）
- **评估工具**
  - LangSmith：全流程追踪 + 评估
  - MLflow：实验管理
  - DeepEval：自动化评估框架
  - Ragas：RAG 评估指标
- **迭代优化流程**
  ```
  问题收集 → 根源分析 → 优化方案 → 实验验证 → 上线监控
  ```
- **监控指标**
  - 用户满意度 / 问题解决率
  - Token 消耗趋势
  - 工具调用成功率
  - 检索质量（Precision / Recall / NDCG）
